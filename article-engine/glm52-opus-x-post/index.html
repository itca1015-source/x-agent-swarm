The Model Is Becoming a Replaceable Part

I thought this was going to be another model fight.

GLM-5.2 vs Opus. Forty demos. Some pretty charts. A winner at the end.

Then I noticed the weird part.

GLM was running inside Claude Code.

Not next to it. Inside it.

The tool was Anthropic's. The interface was Anthropic's. The workflow was Anthropic's.

But the brain underneath was GLM.

That is the part that should make closed-model companies nervous.

For years, the comforting story was simple: users are sticky.

They like the product. They like the workflow. They get used to the shortcuts, the buttons, the way the tool feels.

So even if another model gets close, it does not matter. The experience is the moat.

But what if the experience can just swap brains?

That is what Nick Saraev showed.

He pointed Claude Code at GLM's endpoint using anthropic_base_url. The shell stayed the same. The model changed.

Small trick. Big implication.

People may love Claude Code. That does not mean they are loyal to the exact model behind Claude Code.

The shell is one layer. The brain is another.

And once people really feel that split, the model starts looking less like the product.

It starts looking like a part.

A very important part, yes. But still a part.

Now the model test matters.

Nick ran Opus 4.8 and GLM-5.2 through more than forty practical tasks: 3D scenes, dashboards, explainers, landing pages, small games.

The interesting choice: he did not grade them by benchmarks.

Because at that level, the benchmark scores are already too close to be useful.

That alone is a little strange. We spent the last few years treating leaderboards like sports scores.

Model A is up 0.4 points. Model B is cooked. New king. New discourse. Repeat forever.

But when the scores flatten, the scoreboard stops helping.

So Nick judged taste.

Does the output feel right? Is it readable? Is it restrained? Did the model know when to stop?

This is where GLM surprised me.

In the galaxy demo, Opus made the thing so bright it was almost blown out. GLM's version was cleaner and easier to control.

In the rainbow explainer, GLM's layout felt calmer. Opus looked like it was trying a little too hard.

Same pattern in terrain, landing pages, and other visual tasks.

GLM had better taste more often than I expected.

But not everywhere.

In the block-stacking game, GLM lost. The game ran, but it moved too fast and fell apart almost immediately.

Opus made the more playable version.

That split is useful.

Opus looked stronger on logic and constraints. GLM looked stronger on taste and feel.

That is not a clean sweep. It is more interesting than a clean sweep.

Now add price.

This is where the story stops being cute.

If one model is a little worse in some places, a little better in others, and costs a fraction of the price, the buyer's brain changes.

Nick framed GLM as roughly one fifth the cost of Opus for this kind of work.

That is not a discount. That is a weapon.

You can get to it through z.ai. Or through OpenRouter. Or through hosted providers like Fireworks and DeepInfra.

The more places you can buy the same intelligence, the harder it is to keep the price high.

At some point, cheap stops meaning worse.

It starts meaning dangerous.

Then there is the ownership part.

This is the piece most people skip because it sounds less fun than model drama.

But it matters more.

The full GLM-5.2 is huge: 700 billion parameters. You are not casually running that on a laptop.

But quantized versions change the question.

Nick points to a 2-bit version that cuts the model down dramatically, keeps much of the accuracy, and can fit on a 256 GB Mac.

The important line is simple: if the model runs on your machine, nobody can take it away without taking the machine too.

An API can disappear. Pricing can change. A feature can get policy'd out of existence.

A local model is not magic. It can be slower. It can be worse. It can be annoying.

But it belongs to you in a way rented intelligence does not.

So what is the moat now?

It is not gone. That would be too easy.

It is moving.

If models get close enough, open enough, and cheap enough, the fight moves outside the model.

It moves to taste. Price. Reliability. Latency. Ownership. UX.

It moves to the thing around the model.

That is why Claude Code matters in this story.

The product people love may be the harness, not the model.

And if the harness can swap models, the model provider has a problem.

The practical takeaway is boring, which usually means it is useful.

Do not marry one remote brain.

Use the strongest cheap model that works. Keep another one ready. And if a workflow is truly core to your business, keep a path to something you can run yourself.

Now, the cold water.

Taste is subjective. Nick's prompts were lightweight. Another person could run another set of tasks and get a different result.

GLM losing the game demo matters too. Logic and constraints are not side quests. They are the work.

And open models still come with friction. In that setup, GLM needed outside tools for search.

So no, this does not mean closed models are dead.

It means the story changed.

The wall did not fall over.

Someone found a door in it.
